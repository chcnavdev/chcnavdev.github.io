    Quick Start #

    Permissions #

    Before using the SDK, make sure that the application has the necessary permissions. These permissions include Bluetooth (only Bluetooth connections are used in this section and WIFI is not involved), and the ability to read and write external storage, as the SDK requires external storage to read and write configuration files and log files. Please add permissions in AndroidManifest.xml.

    <!-- Permission to use Bluetooth features -->
<uses-permission android:name="android.permission.BLUETOOTH"/>
<!-- Permission to administer Bluetooth settings -->
<uses-permission android:name="android.permission.BLUETOOTH_ADMIN"/>
<!-- Permission to read from external storage -->
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"/>
<!-- Permission to write to external storage -->
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>

    It should be noted that in Android10, Android11 and later versions, there will be some differences in the application method for external storage. Android 10 introduces Scoped Storage, You can add the following code to AndroidManifest.xml:

    <application
    android:requestLegacyExternalStorage="true"
    ... >

    The requestLegacyExternalStorage tag is completely abandoned in Android 11. All applications must follow the Scoped Storage model. At the same time, a new special permission is introduced, which allows applications to obtain permissions similar to traditional global file access. The specific declaration code is as follows:

    <uses-permission android:name="android.permission.MANAGE_EXTERNAL_STORAGE"/>

    At the same time, the code needs to guide the user to manually grant permissions in the system settings:

    Intent intent = new Intent(Settings.ACTION_MANAGE_ALL_FILES_ACCESS_PERMISSION);
startActivity(intent);

    SDK Initialization #

    After obtaining permissions, initialize the GNSS service and start monitoring the connection status. The following are the initialization steps:

    new Thread(new Runnable() {
    @Override
    public void run() {
        try {
            // Initialize the Service using ApplicationContext and resource file path parameter
            GnssServiceHelper.getInstance().start(App.getInstance().getApplicationContext(), SdcardUtils.getAppFolder());
            // Get the Service instance
            mService = GnssServiceHelper.getInstance().getGnssService();
            // Receiver data callback
            mService.requestReceiverListener(mReceiverListener);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}).start();

    App.getInstance().getApplicationContext() is an application-level Context, and SdcardUtils.getAppFolder() can be any path with read and write permissions. mService.requestReceiverListener(mReceiverListener) is mainly used to set data callbacks, such as coordinate data, etc. The specific implementation of the class to which the mReceiverListener object belongs is described in Obtain data.

    Connected devices #

    SDK provides you two ways to connect to the device, respectively, Bluetooth connection and WIFI connection, of which Bluetooth connection provides you two ways: through the interface provided by the SDK connection, as well as through the injection method to realize Bluetooth connection and disconnection, here to let you get started quickly, we only introduces the way to realize Bluetooth connection directly through the SDK, the other ways to connect. For other connection methods, please refer to “Chapter 2”, the main steps of connection are as follows:

    1. Configure the connection parameters via ReceiverConnectionOption;
    2. Get the IReceiverConnectManager object from the service obtained in SDK Initialization;
    3. Pass the ReceiverConnectionOption object obtained in “1” as a parameter to the connect method of IReceiverConnectManager object.
    // Connection event
mBTConnect.setOnClickListener(new View.OnClickListener() {
    @Override
    public void onClick(View view) {
        // Connect directly through the SDK
        ReceiverConnectionOption option = new ReceiverConnectionOption()
                .setConnectionType(ConnectionType.CONNECTION_BLUETOOTH)
                .setProductName("SMART-RTK")
                .setBluetoothName("GNSS-9999848");
       
        try {
            // Get receiverConnectManager through mService and then connect using receiverConnectManager
            IReceiverConnectManager receiverConnectManager = IReceiverConnectManager
                    .Stub.asInterface(mService.getReceiverConnectManager());
            // Connect using receiverConnectManager
            if (!receiverConnectManager.connect(option)) {
                // Connection failure
            }
        } catch (RemoteException e) {
            throw new RuntimeException(e);
        }
    }
});

    Disconnect #

    Get the IReceiverConnectManager object through the service obtained in SDK Initialization, and then disconnect the device by calling the disconnect method of the IReceiverConnectManager object. The specific code is as follows:

    // Disconnection event
mBTDisConnect.setOnClickListener(new View.OnClickListener() {
    @Override
    public void onClick(View view) {
        try {
            IReceiverConnectManager receiverConnectManager = IReceiverConnectManager
                    .Stub.asInterface(mService.getReceiverConnectManager());
            if (receiverConnectManager != null) {
                receiverConnectManager.disConnect();
            }
        } catch (RemoteException e) {
            throw new RuntimeException(e);
        }
    }
});

To make it easier for you to get started quickly, we only introduce how to obtain coordinate data through this callback. For more implementations, please refer to Chapter 2.

    }

}
